# 태그 분류 체계 (대/중/소)

**관련 태스크**: A5
**개정**: 2026-07-22 — 중분류를 2축(형식/대상) 닫힌 enum으로 재설계. 배경: eval 실측(F1 0.746 회귀·exact 0.49)에서
주제형 중분류(LLM·프론트엔드)와 형식형 중분류(강의)가 같은 자리를 다퉈 gold조차 비일관(강의/LLM 혼동 23건).
중분류를 닫힌 어휘로 고정해 모델 자유 생성(환각)을 구조적으로 차단.

---

## 구조

태그는 대→중→소 계층으로 최대 3개 생성.

```
대분류 (주제 영역) → 중분류 (닫힌 enum: 형식 또는 대상) → 소분류 (구체 고유명사·자유 텍스트)
```

| 태그 수 | 조건 |
|---------|------|
| 0개 | 내용 파악 불가(로그인·오류·광고), 또는 13개 대분류 밖 주제(종교·법률·임상의료·정치·프로스포츠·반려동물 등) → 미분류 |
| 1개 | 대분류만 명확 |
| 2개 | 대+중 식별 (소가 중과 동일하거나 자명할 때) |
| 3개 | 대+중+소 모두 명확히 다른 정보 |

---

## 중분류 2축 원칙 (핵심)

대분류는 성격에 따라 두 부류로 나뉘고, **중분류는 부류별 닫힌 enum에서만 선택**한다. 자유 생성 금지.

| 부류 | 대분류 | 중분류 축 |
|------|--------|-----------|
| **콘텐츠형** | 개발 · AI/ML · 디자인 · 비즈니스 · 학습 · 콘텐츠 · 게임 | **행동 양식(형식)** — 공통 enum |
| **대상형** | 커뮤니티 · 브랜드 · 라이프스타일 · 여행 · 금융 · 쇼핑 | **대상** — 대분류별 enum |

- 주제 세분(프론트엔드·백엔드·LLM·RAG·UI/UX·스타트업 등 구 중분류)은 **중분류 금지 → 소분류로 강등**.
  예: `개발>프론트엔드>Next.js` (구) → `개발>아티클>Next.js` 또는 `개발>공식문서>Next.js` (신).
- 근거: "AI 코딩툴 강의"에서 형식축(강의)과 주제축(LLM)이 중분류 한 자리를 다투는 구조 모순 제거.
  같은 대상의 강의·문서·도구가 소분류로 묶여 검색·필터 일관성 확보.

## 대분류 결정 (주제 우선)

- 대분류 = **주제**. 형식은 중분류가 담당.
  - 개발 강의 영상 → `개발>강의`, AI툴 사용법 영상 → `AI/ML>강의`, 디자인 아티클 → `디자인>아티클`
- **학습**: 주제가 12개 도메인 밖(수학·어학 등)이거나, 커리큘럼형 코스·부트캠프(다회차 시리즈·완주반).
  단발 how-to 영상은 주제 도메인 대분류(예: 단발 Codex 활용 영상 = `AI/ML>강의`, Next.js 30강 완주반 = `학습>강의>Next.js`).
- **콘텐츠**: 도메인 특정 없는 일방향 읽기·구독물만. 주제가 도메인이면 도메인 우선.
- **커뮤니티**: 사용자 간 상호작용(게시·댓글·질문·팔로우)이 핵심 기능일 때만. 게임 커뮤니티는 `게임>포럼`.
- **브랜드**: 기업·제품 브랜드 소개·마케팅·캠페인. 구매 액션 페이지는 쇼핑.
- **게임**: 공략·뉴스·리뷰·e스포츠·게임 커뮤니티. 게임 "개발"(Unity·Unreal)은 개발.

## 생성 규칙

1. **순서 고정**: 항상 대→중→소 순으로 출력
2. **중분류 enum 강제**: 위 표의 어휘만 허용 — 목록 밖 중분류 생성 금지 (자동화·LLM·프론트엔드 등은 소분류로)
3. **중복 생략**: 중분류와 소분류가 같은 의미면 소분류 제거
4. **플랫폼명 소분류 금지**: 유튜브·깃허브 등 플랫폼은 태그 아님 — 영상·페이지 주제로 대분류 결정
5. **소분류는 고유명사 우선**: 추상어보다 기술명·브랜드명. 제목에 명시된 고유명(헤르메스·게임명 등)은 반드시 포함, 명시 안 된 툴명 추론 삽입 금지

### 경계 규칙 (라이프스타일·여행·금융)

- **레시피 vs 맛집**: 직접 조리(레시피)=`라이프스타일>요리`. 가게·장소 정보=`여행>맛집`.
- **식품**: 식료품 구매=`쇼핑>식품`. 레시피는 `라이프스타일>요리`.
- **자동차**: 차량 정보·관리·전기차=`라이프스타일>자동차`. 단순 구매 액션 페이지=`쇼핑`.
- **부동산**: 거주·투자 목적 구분 없이 모두 `금융>부동산`.
- **건강**: 영양·수면·웰니스=`라이프스타일>건강`. 증상·질병·약·진단(임상)은 도입 보류 → 확신 없으면 0태그.

### 경계 규칙 (비즈니스·디자인)

- **기획/PM**: 제품 기획·서비스 구체화·요구사항 정의·PM·로드맵·우선순위 결정 = `비즈니스`(형식은 강의/아티클, 산출물명 PRD 등은 소분류). **디자인으로 흡수 금지** — UI/그래픽 실무가 주제일 때만 디자인.
  - 배경: `youtube.com/watch?v=1esnmymO3cw`("기능 구현 단계 서비스 구체화 기준")가 디자인으로 오분류됨. "서비스 구체화"가 디자인과 어휘적으로 가까워 최근접 흡수. 기획 대분류 부재 + 경계 규칙 부재가 원인. v2 형식축 유지 위해 기획은 소분류로 두고 경계 규칙으로 대분류만 교정.

### 경계 규칙 (AI/ML·개발)

- **AI 코딩툴 방법론**: Claude Code·Codex·Cursor·헤르메스 등 AI 코딩 에이전트의 **활용법·에이전트·하네스·워크플로우·스킬·프롬프트 설계**가 주제면 `AI/ML`(코딩툴을 써도 AI 방법론이 주제면 AI/ML). **ML·데이터분석 라이브러리·추론엔진**(OpenCV·pandas·vLLM 등)도 `AI/ML`.
  - 예외: 그 툴로 **특정 언어/프레임워크 구현·앱 배포** 등 구체적 개발 산출물이 주제면 그 도메인(n8n으로 GitHub 자동화=`개발`). **DB·인프라·언어/프레임워크**(PostgreSQL·Kubernetes·React 등)는 `개발`.
  - 배경(2026-07-24, S1): 실측 최대 오분류 버킷 = AI/ML↔개발(하네스·코덱스 강의·OpenCV·pandas 등이 개발로 흡수). 경계 규칙 도입으로 **rich macro-F1 0.757→0.768, 대분류 정확도 0.840→0.859(+0.019)**. 도입 중 PostgreSQL이 "데이터 라이브러리"로 AI/ML에 흡수되던 회귀 1건 발견 → DB/인프라 예외구로 교정.
### 경계 규칙 (학습 — 주제 우선, 2026-07-25 확정)

- **강의/코스는 주제의 대분류로**: 커리큘럼형 코스·부트캠프·완주반도 주제가 특정 도메인이면 그 도메인 대분류(형식은 중분류 `강의`) — Next.js 완주반=`개발`, ML 코스=`AI/ML`, 비즈니스 스쿨=`비즈니스`. 형식(강의)은 중분류 형식축이 이미 담당하므로 대분류까지 형식으로 잡으면 축이 중복된다.
- **`학습` 대분류는 주제가 12개 도메인 밖일 때만**: 수학·어학·자격증 시험 안내·학습법 등 (Khan Academy 수학, 큐넷 정보처리기사 시험 안내).
- 배경: gold 15건이 형식 우선(강의→학습)으로 라벨돼 있었는데 모델은 일관되게 주제를 선호 — AI/ML 경계 규칙("Claude Code 강의→AI/ML")과도 충돌. 주제 우선으로 통일하고 gold 12건 재라벨(학습 잔류 3건: Khan·큐넷·개인 학습 문서).

## 보류 대분류 (미도입)

일반 분류 제안에 있었으나 현재 도입 보류. 사유:

| 후보 | 보류 사유 |
|------|----------|
| 의료/건강관리 (임상) | 증상·약·진단 오분류 시 안전 리스크 큼. 웰니스 수준은 `라이프스타일>건강`으로 흡수. 임상 콘텐츠는 확신 없으면 0태그. |
| 법률/계약 | 금융·비즈니스와 경계 모호 → 단일 대분류 곤란. 전문 콘텐츠 표본 부족. |
| 뉴스/시사 | 주제가 아닌 형식(시의성) 축 → 형식은 중분류 enum("뉴스")이 흡수. 대분류 신설 불필요. |

공통: 현 데이터셋(디자인·PM·AI 북마크)에 표본 ≈0 → 골든셋으로 정확도 측정 불가. 실데이터 유입 후 재검토.

## 크롬 북마크 import — folder_hint

### 개요

크롬 HTML 내보내기 시 폴더 구조를 두 가지로 분리 처리:

| 구분 | 처리 |
|------|------|
| 크롬 기본 폴더 (북마크 바·다른 북마크·모바일 북마크) | 무시 — 카테고리·folder_hint 미포함 |
| 사용자 폴더 | `folder_hint`에 경로 배열로 저장 |

### category_id vs folder_hint

- `category_id` — AI `tags[0]` 기준. 폴더명과 무관.
- `folder_hint` — 원본 크롬 폴더 경로. `category_id`와 별개의 연결고리.

```
크롬 경로: 북마크 바 > 개발 > 프론트엔드
→ 기본 폴더 제거: ["개발", "프론트엔드"]
→ folder_hint: ["개발", "프론트엔드"]
→ category_id: tags[0] ("개발" → categories 조회)
```

### 경우별 처리

| 경우 | folder_hint | category_id |
|------|------------|-------------|
| 사용자 폴더 내 북마크 | `["개발", "프론트엔드"]` | `tags[0]` 매핑 |
| 기본 폴더 직속 북마크 | `null` | `tags[0]` 매핑 |
| 태깅 실패 (`tags = []`) | 폴더 경로 그대로 | `null` (미분류) |

```
  대상형: 커뮤니티(포럼·Q&A·소셜미디어) 브랜드(마케팅·기업) 라이프스타일(운동·요리·인테리어·육아·건강·자동차) 여행(숙소·항공·관광·맛집) 금융(투자·부동산·세금·보험) 쇼핑(전자기기·소프트웨어·패션·식품·기타용품)
소분류: 구체 고유명사 (예: Next.js, 헤르메스, ChatGPT)

## 태그 수 규칙
- 0개: 내용 파악 불가(로그인·오류·광고), 또는 13개 대분류 어디에도 안 맞는 주제 → 억지 태깅 금지(미분류)
- 1개: 대분류만 명확 / 2개: 대+중 / 3개: 대+중+소

## 출력
JSON만 반환. 각 태그에 0~1 confidence(0.6 미만은 자동 제외).
{"tags": [{"tag": "대분류", "confidence": 0.95}, {"tag": "중분류", "confidence": 0.8}]}
```

> 위는 요약. SYSTEM_PROMPT 전문·경계 규칙·Few-shot 예제는 `lib/ai.ts` 단일 출처(이 블록과 드리프트 시 코드 우선).
