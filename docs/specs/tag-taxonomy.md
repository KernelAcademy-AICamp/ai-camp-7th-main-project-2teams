# 태그 분류 체계 (평면 태그 + 카테고리)

**관련 태스크**: A5

---

## 구조

두 축으로 분리. 계층(대/중/소) 없음.

```
category (단일 대분류, 1개)  +  tags (평면 라벨, 0~3개, 순서 무관)
```

| 축 | 역할 | 개수 | 출처 |
|----|------|------|------|
| `category` | 단일 분류 (사이드바 1차 필터) | 1개 (12종 중 1 또는 미분류) | AI 직접 지정 |
| `tags` | 자유 다중 라벨 (교차 검색) | 0~3개 | AI 생성, 정규화 |

- `category`는 `tags`와 **독립** — 더 이상 `tags[0]`에 종속되지 않음.
- `tags`는 동등한 평면 라벨 — 분야(프론트엔드·RAG)와 고유명사(Next.js·Reddit)를 순서 무관하게 나열.
- 키워드 없으면 `tags=[]`, 분류 불가면 `category="미분류"`.

> 아래 "분류 트리"는 계층이 아니라 **카테고리별 권장 태그 어휘집**(few-shot 참고용).

## 생성 규칙

1. **category 1개**: 12종 중 가장 잘 맞는 하나, 불가 시 "미분류"
2. **tags는 평면**: 순서 무관, 동등한 라벨 — 계층 없음
3. **카테고리명 중복 금지**: `category`와 동일한 값을 `tags`에 넣지 않음
4. **상위어 금지**: 포함 관계면 더 구체적인 것만
5. **URL 자명 정보 생략**: 플랫폼명 등 URL로 자명하면 태그 생략
6. **태그는 고유명사 우선**: 추상어보다 기술명·브랜드명

### 경계 규칙 (라이프스타일·여행·금융)

- **레시피 vs 맛집**: 직접 조리(레시피)=`라이프스타일>요리`. 가게·장소 정보=`여행>맛집`.
- **식품**: 식료품 구매=`쇼핑>식품`. 레시피는 `라이프스타일>요리`.
- **자동차**: 차량 정보·관리·전기차=`라이프스타일>자동차`. 단순 구매 액션 페이지=`쇼핑`.
- **부동산**: 거주·투자 목적 구분 없이 모두 `금융>부동산`.
- **건강**: 영양·수면·웰니스=`라이프스타일>건강`. 증상·질병·약·진단(임상)은 도입 보류 → 확신 없으면 0태그.

## 보류 대분류 (미도입)

일반 분류 제안에 있었으나 현재 도입 보류. 사유:

| 후보 | 보류 사유 |
|------|----------|
| 의료/건강관리 (임상) | 증상·약·진단 오분류 시 안전 리스크 큼. 웰니스 수준은 `라이프스타일>건강`으로 흡수. 임상 콘텐츠는 확신 없으면 0태그. |
| 법률/계약 | 금융·비즈니스와 경계 모호 → 단일 대분류 곤란. 전문 콘텐츠 표본 부족. |
| 뉴스/시사 | 주제가 아닌 형식(시의성) 축 → 주제 기반 대분류와 직교, 단일 분류 충돌. |

공통: 현 데이터셋(디자인·PM·AI 북마크)에 표본 ≈0 → 골든셋으로 정확도 측정 불가. 실데이터 유입 후 재검토.

## 크롬 북마크 import — folder_hint

### 개요

크롬 HTML 내보내기 시 폴더 구조를 두 가지로 분리 처리:

| 구분 | 처리 |
|------|------|
| 크롬 기본 폴더 (북마크 바·다른 북마크·모바일 북마크) | 무시 — 카테고리·folder_hint 미포함 |
| 사용자 폴더 | `folder_hint`에 경로 배열로 저장 |

### category_id vs folder_hint

- `category_id` — AI가 직접 지정한 `category` 기준. `tags`·폴더명과 무관.
- `folder_hint` — 원본 크롬 폴더 경로. `category_id`와 별개의 연결고리.

```
크롬 경로: 북마크 바 > 개발 > 프론트엔드
→ 기본 폴더 제거: ["개발", "프론트엔드"]
→ folder_hint: ["개발", "프론트엔드"]
→ category_id: AI category ("개발" → categories 조회), tags와 독립
```

### 경우별 처리

| 경우 | folder_hint | category_id |
|------|------------|-------------|
| 사용자 폴더 내 북마크 | `["개발", "프론트엔드"]` | AI `category` 매핑 |
| 기본 폴더 직속 북마크 | `null` | AI `category` 매핑 |
| 분류 실패 (`category="미분류"`) | 폴더 경로 그대로 | `null` (미분류) |

## 시스템 프롬프트

```
당신은 북마크 분류기입니다. 웹페이지에 카테고리(category) 1개와 평면 태그(tags) 0~3개를 부여합니다.

## 카테고리 (category) — 정확히 1개
12종 중 가장 잘 맞는 하나. 판단 불가(로그인·오류·광고)면 "미분류".
개발 · AI/ML · 디자인 · 비즈니스 · 학습 · 쇼핑 · 커뮤니티 · 브랜드 · 게임 · 라이프스타일 · 여행 · 금융

## 태그 (tags) — 평면, 0~3개, 순서 무관
분야(프론트엔드·RAG·요리) + 구체 기술명·고유명사(Next.js·Reddit·Nike). 계층 없음.
- 카테고리명과 동일한 태그 금지(중복)
- URL로 자명한 정보(플랫폼명)는 태그 생략

## 출력
JSON만 반환. category 1개 + tags 배열(각 {tag, confidence}). 0.6 미만 태그는 자동 제외.
{"category": "대분류", "tags": [{"tag": "키워드", "confidence": 0.85}]}
```

> 위는 요약. SYSTEM_PROMPT 전문·경계 규칙·Few-shot 예제는 `lib/ai.ts` 단일 출처(이 블록과 드리프트 시 코드 우선).
