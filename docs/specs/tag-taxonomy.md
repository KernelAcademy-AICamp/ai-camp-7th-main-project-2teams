# 태그 분류 체계 (대/중/소)

**관련 태스크**: A5

---

## 구조

태그는 대→중→소 계층으로 최대 3개 생성.

```
대분류 (영역) → 중분류 (분야) → 소분류 (구체 주제)
```

| 태그 수 | 조건 |
|---------|------|
| 0개 | 내용 파악 불가 (로그인 필요·오류·광고 페이지) |
| 1개 | 대분류만 명확 |
| 2개 | 대+중 식별 (소가 중과 동일하거나 자명할 때) |
| 3개 | 대+중+소 모두 명확히 다른 정보 |

## 생성 규칙

1. **순서 고정**: 항상 대→중→소 순으로 출력
2. **상위어 금지**: 대+중이 포함 관계면 더 구체적인 것만 (`AI` + `LLM` → `LLM` 불가, 대분류는 `AI/ML` 유지)
3. **중복 생략**: 중분류와 소분류가 같은 의미면 소분류 제거
4. **URL 자명 정보 생략**: `youtube.com` → `강의` 소분류 불필요
5. **소분류는 고유명사 우선**: 추상어보다 기술명·브랜드명

## 크롬 북마크 import — folder_hint

### 개요

크롬 HTML 내보내기 시 폴더 구조를 두 가지로 분리 처리:

| 구분 | 처리 |
|------|------|
| 크롬 기본 폴더 (북마크 바·다른 북마크·모바일 북마크) | 무시 — 카테고리·folder_hint 미포함 |
| 사용자 폴더 | `folder_hint`에 경로 배열로 저장 |

### category_id vs folder_hint

- `category_id` — AI `tags[0]` 기준. 폴더명과 무관.
- `folder_hint` — 원본 크롬 폴더 경로. `category_id`와 별개의 연결고리.

```
크롬 경로: 북마크 바 > 개발 > 프론트엔드
→ 기본 폴더 제거: ["개발", "프론트엔드"]
→ folder_hint: ["개발", "프론트엔드"]
→ category_id: tags[0] ("개발" → categories 조회)
```

### 경우별 처리

| 경우 | folder_hint | category_id |
|------|------------|-------------|
| 사용자 폴더 내 북마크 | `["개발", "프론트엔드"]` | `tags[0]` 매핑 |
| 기본 폴더 직속 북마크 | `null` | `tags[0]` 매핑 |
| 태깅 실패 (`tags = []`) | 폴더 경로 그대로 | `null` (미분류) |

## 시스템 프롬프트

```
당신은 북마크 분류기입니다. 웹페이지를 대/중/소 계층으로 분류해 태그 0~3개를 생성합니다.

## 분류 체계
대분류: 개발 · AI/ML · 디자인 · 비즈니스 · 학습 · 쇼핑 · 커뮤니티 · 브랜드 · 게임
중분류: 대분류 하위 분야 (예: 개발→프론트엔드, AI/ML→RAG)
소분류: 구체적 기술명·고유명사 (예: Next.js, pgvector, 프롬프트엔지니어링)

## 태그 수 규칙
- 0개: 내용 파악 불가 (로그인·오류·광고 페이지)
- 1개: 대분류만 명확
- 2개: 대+중 식별 (소가 중과 동일하거나 자명할 때)
- 3개: 대+중+소 모두 명확히 다른 정보일 때

## 출력 규칙
- 순서: 항상 대→중→소
- URL로 자명한 정보(플랫폼명 등)는 소분류 생략
- 중분류와 소분류가 같으면 소분류 생략

## 출력
JSON만 반환. 설명 없음.
{"tags": ["대분류", "중분류", "소분류"]}
```
