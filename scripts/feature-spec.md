## 2. 입력 검증 기준값 (코드 근거)

기능 정의와 엣지케이스 판단의 기준선. `front/lib/schemas.ts` 실코드 기준.

| 필드 | 제약 | 위반 시 |
|---|---|---|
| `title` | 1~500자 | 400 |
| `url` | `z.url()` 형식 | 400 |
| `content` | 0~2000자 | 400 |
| 검색 `query` | 1~50자 | 400 |
| `tags` | 배열 최대 10개, 항목당 1~50자 | 400 |
| `category` | 1~50자 | 400 |
| `description` | 최대 2000자 | 400 |
| 임포트 파일 크기 | 최대 5MB | 413 |
| 임포트 항목 수 | 최대 500개(초과분 skip) | — |
| 썸네일 이미지 | 최대 5MB | 502 |
| `(user_id, url)` | UNIQUE | 409 |

## 3. 유스케이스 시나리오

기본흐름/예외흐름은 `scripts/requirements-definition.md` §9(SC-01~SC-11) 원본 유지. 본 문서는 각 시나리오에 **엣지케이스**를 추가 매핑한다.

### SC-01 비로그인 방문
액터: 비로그인 방문자. `/welcome` → 서비스 소개 → 인증상태 확인 → 로그인 CTA.

### SC-02 신규가입 온보딩
액터: 신규 사용자. OAuth 로그인 → `/onboarding` → 익스텐션 설치 안내 → 첫 저장 유도(Aha 모멘트).

### SC-03 익스텐션 북마크 저장
액터: 로그인 사용자. 단축키/버튼 → 탭 정보 수집(제목/URL/본문 2000자) → `POST /api/bookmarks` → AI 태깅+임베딩 → 토스트.

### SC-04 웹앱 URL 직접 추가
액터: 로그인 사용자. 헤더 버튼 → URL 모달 → SC-03과 동일 처리.

### SC-05 자연어 검색
액터: 로그인 사용자. 검색어(≤50자) → 임베딩 변환 → pgvector+pg_trgm RRF → 상위 20건(threshold 0.5).

### SC-06 필터 조합 탐색
액터: 로그인 사용자. 카테고리/즐겨찾기 탭 + 검색어 + 태그 필터 조합, 정렬/뷰 전환.

### SC-07 파일 임포트
액터: 로그인 사용자. 소스 선택(HTML/카카오CSV) → 업로드 → 검증 → AI 태깅 → 성공/실패/부분완료 분기.

### SC-08 북마크 편집·삭제
액터: 로그인 사용자. 카드 메뉴 → 삭제 즉시 처리 / 수정 시 태그·카테고리·설명 편집(재임베딩).

### SC-09 죽은 링크 감지
액터: 시스템(저장 시점 자동). HEAD→GET 폴백 → 404/410만 `is_dead=true` → 배지 표시.

### SC-10 내보내기 → 재임포트 왕복 복원
액터: 로그인 사용자. HTML/CSV 내보내기 → 자체 포맷 재업로드 → TAGS 존재 시 재태깅 생략.

### SC-11 개인정보 열람 / 회원탈퇴
액터: 로그인 사용자. `GET /api/account`(열람) / `DELETE /api/account`(탈퇴, CASCADE 원자적 처리) → 익스텐션 로컬 데이터 파기.

## 4. 엣지케이스

시나리오별로 정상흐름 밖의 입력·상태·경합 조건을 정리. "확인 필요"는 현재 명세·코드에 정의가 없어 기획/개발 확인이 필요한 항목.

### 4.1 SC-03/04 — 북마크 저장 (AI 태깅)

| 케이스 | 예상동작 / 근거 |
|---|---|
| URL 형식 위반(`javascript:`, 빈 문자열) | `z.url()` 검증 실패 → 400 |
| title 500자 초과 | 400 |
| content 2000자 초과 전송 | 400 (클라 collector는 2000자 슬라이스로 원천 차단) |
| 동일 URL 재저장 | UNIQUE 제약 → AI 호출 전 409 선검사, 저장 전 경고 UI |
| 동일 URL 동시 2탭 저장(레이스) | 선검사 통과 후 경합 시 UNIQUE 위반 catch로 409 |
| OpenAI 태깅 API 타임아웃/장애 | `tags=[]` → `category_id NULL` → '미분류' |
| 임베딩 생성 실패 | 502, bookmarks row 미생성 |
| 페이지 본문 없음(SPA/JS 렌더) | title 단독 임베딩 + `weak-vector` 경고 로그(A37) |
| 302/301 리다이렉트 체인 URL | 원본 URL 그대로 저장(리다이렉트 미추적) — **확인 필요** |
| non-http 스킴(`chrome://`, `file://`) 탭에서 저장 시도 | z.url() 통과 여부 미확정, 익스텐션 버튼 활성/비활성 처리 — **확인 필요** |

### 4.2 SC-05 — 자연어 검색

| 케이스 | 예상동작 / 근거 |
|---|---|
| 빈 문자열 검색 | 400 |
| 50자 초과 | 400 (클라 입력 제한 + 서버 이중 검증) |
| SQL 인젝션 패턴(`'; DROP TABLE`) | 파라미터 바인딩 경유 — 인젝션 불가, 트라이그램 매칭 노이즈만 가능 |
| 검색 결과 0건 | 빈 상태 UI |
| 필터 조합 + 검색 결과 0건 | "필터 탓 0건"과 "검색어 자체 무결과" 구분 여부 — **확인 필요** |
| 이모지/한자 검색어 | 임베딩 모델 지원 범위 내로 추정, 실측 없음 — **확인 필요** |

### 4.3 SC-07 — 파일 임포트

| 케이스 | 예상동작 / 근거 |
|---|---|
| 5MB 초과 파일 | 413 |
| 500개 초과 항목 | 초과분 `skipped` 카운트로 잘림 — UI 노출 여부 **확인 필요** |
| 잘못된 MIME/확장자 위장 | 400 |
| 파일 파싱 자체 실패(깨진 HTML/CSV) | 400 |
| 업로드 중 취소 | 서버 처리 이미 시작된 항목의 롤백 여부 — **확인 필요** |
| 파일 내부 중복 URL | DB UNIQUE로 후단 차단되나 스킵/실패목록 처리 방식 — **확인 필요** |
| 소스 선택-실제 파일 형식 불일치 | 파싱 실패 400으로 귀결 추정, "소스 재선택" 안내 여부 — **확인 필요** |

### 4.4 SC-08 — 편집/삭제

| 케이스 | 예상동작 / 근거 |
|---|---|
| 잘못된 id 형식 | 400 |
| 존재하지 않는/타인 소유 id | RLS 필터 후 404 |
| tags 11개 이상 | 400 |
| tag 1개 51자 이상 | 400 |
| description 2000자 초과 | 400, 재임베딩 트리거 전 차단 |
| 삭제 직후 동일 카드 더블클릭(레이스) | 두 번째 요청 404 처리 여부 — **확인 필요** |

### 4.5 SC-09 — 죽은 링크 감지

| 케이스 | 예상동작 / 근거 |
|---|---|
| 403/429/5xx/timeout | 죽은 링크 미판정(사이트 생존 가능성) |
| 404/410 | `is_dead=true` |
| HEAD 미지원 서버 | GET 폴백 |
| 과거 저장분(기능 배포 이전 데이터) | 자동 소급 없음, `backfill-dead-link.ts` 수동 실행 |

### 4.6 SC-11 — 탈퇴/인증

| 케이스 | 예상동작 / 근거 |
|---|---|
| 탈퇴 처리 중 세션 만료 | 401, 탈퇴 미완료 상태 재시도 UX — **확인 필요** |
| 탈퇴 API 500(부분 실패) | CASCADE 원자적 처리로 전체 롤백 추정(A32) |
| 익스텐션 로그인 상태에서 웹앱 탈퇴 | 익스텐션 세션 무효화는 로그아웃 시점에만 로컬 파기 — 탈퇴 즉시 반영 여부 **확인 필요** |

### 4.7 크로스커팅 (전 시나리오 공통)

| 케이스 | 예상동작 / 근거 |
|---|---|
| 세션 만료 상태로 API 호출 | `withAuth` 401, 화면별 재로그인 리다이렉트 일관성 — **확인 필요** |
| RLS 우회 시도(타 유저 id 직접 호출) | RLS 원천 차단, 응답 404/빈 배열 |
| 네트워크 끊김 중 저장 버튼 연타 | 클라이언트 debounce/disable 여부 — **확인 필요** |
| 모달 오픈 중 페이지 이탈/새로고침 | 진행 중 작업(임포트 업로드 등) 유실 여부 — **확인 필요** |

## 5. 미해결 항목 요약

"확인 필요" 표시된 항목은 기획/개발 확인 후 `scripts/requirements-definition.md` 시나리오 예외흐름에 반영 대상. 우선순위 제안:

1. **높음**: 동시성/레이스 계열(동일 URL 동시 저장, 삭제 더블클릭) — 데이터 정합성 직결
2. **중간**: 임포트 취소·부분실패 UX, 세션 만료 재시도 흐름 — 사용자 체감 영향
3. **낮음**: non-http 스킴, 이모지 검색어 — 발생 빈도 낮음
