# IA 설계 — 북마크 AI 관리 서비스 (`ia.xlsx` 변환)

**버전**: v1.0
**작성일**: 2026-07-09
**원본**: `scripts/ia.xlsx` (PRD v0.8 업데이트 기준, 뎁스0~6 트리, 103행)

본 문서는 신규 IA 설계가 아니라 **기존 스프레드시트의 마크다운 변환 + 요구사항정의서(`requirements-definition.md`) SCR-ID 대조**다. `ia.xlsx`는 이미 화면·레이아웃·컴포넌트·요소·인터랙션 6단계 뎁스로 상세히 설계돼 있었다.

## 4. 원본(IA) vs 실 구현 괴리 각주

`ia.xlsx`(PRD v0.8 기준)는 기획 시점 스냅샷이라, 이후 태스크(A58~A65)에서 실 구현이 IA보다 앞서가거나 갈라진 지점이 있다.

1. **검색+필터 동시적용**(§3.3 정렬): IA엔 "v1.1"로만 표기, 실제로는 REQ-F-02(A58)로 done 확인됨 — `requirements-definition.md` §10 추적성 매트릭스 참조.
2. **카드 수정 모달**(§3.3 메뉴버튼): IA엔 "태그·카테고리 직접 편집"까지만 기재, 실 구현(A60)은 설명(description) 필드 편집 + 재임베딩까지 확장 — SCR-10(EditBookmarkModal)이 IA보다 범위가 넓다.
3. **문제 보고**(§3.5 실패상태): IA엔 v1.1 포함 항목으로 기재돼 있으나, `tasks/README.md` A61 상세에는 "문제 보고는 스코프 제외"로 명시 — 미구현 확정, IA 갱신이 필요한 항목이다.
4. **개인 북마크 대시보드**(§3.4): IA·PRD·요구사항정의서 3곳 모두 일치 — 유일하게 설계-구현 갭이 그대로 유지된 항목(A64 pending)이다.
