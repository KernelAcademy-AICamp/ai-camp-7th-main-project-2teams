## 2. 우선순위 확인 필요 항목 (8건) — 먼저 검토

### 높음 — 데이터 정합성 직결

- [ ] **EC-05** (SC-03·04) 동일 URL 동시 2탭 저장(레이스) — 현재: 선검사 통과 후 경합 시 UNIQUE 위반 catch로 409. _재현 테스트로 실제 경합 시나리오 검증 권장._
- [ ] **EC-29** (SC-08) 삭제 직후 동일 카드 더블클릭(레이스) — 두 번째 요청 404 처리 여부 미확인. _결정 필요: 조용히 무시 vs 에러 토스트._

### 중간 — 사용자 체감 영향

- [ ] **EC-21** (SC-07) 파일 임포트 업로드 중 취소 — 서버에서 이미 처리 시작된 항목 롤백 여부. _결정 필요: 부분 저장 허용 vs 전체 롤백._
- [ ] **EC-34** (SC-11) 회원탈퇴 처리 중 세션 만료 — 401 응답 후 재시도 UX 미정의. _결정 필요: 자동 재로그인 유도 문구._

### 낮음 — 발생 빈도 낮음

- [ ] **EC-09** (SC-03·04) 302/301 리다이렉트 체인 URL — 원본 URL 그대로 저장(리다이렉트 미추적). _허용 여부만 확인._
- [ ] **EC-10** (SC-03·04) non-http 스킴(`chrome://`, `file://`) 탭에서 저장 시도 — 익스텐션 버튼 활성/비활성 처리 미정의.
- [ ] **EC-16** (SC-05) 이모지/한자 검색어 — 임베딩 모델 지원 범위 실측 없음.

## 3. 전체 엣지케이스 체크리스트 (40건, 시나리오별)

범례: `[정의됨]` = 코드/정책 근거 있음(참고용 확인) · `[확인 필요]` = 기획/개발 결정 필요

### SC-03 · SC-04 — 북마크 저장 (AI 태깅)

- [ ] EC-01 `[정의됨]` URL 형식 위반(`javascript:`, 빈 문자열) → `z.url()` 검증 실패, 400
- [ ] EC-02 `[정의됨]` title 500자 초과 → 400
- [ ] EC-03 `[정의됨]` content 2000자 초과 전송 → 400(클라 2000자 슬라이스로 원천 차단)
- [ ] EC-04 `[정의됨]` 동일 URL 재저장 → UNIQUE 제약, AI 호출 전 409 선검사 + 경고 UI
- [ ] EC-05 `[확인 필요·높음]` 동일 URL 동시 2탭 저장(레이스) → UNIQUE 위반 catch로 409
- [ ] EC-06 `[정의됨]` OpenAI 태깅 API 타임아웃/장애 → `tags=[]` → '미분류'
- [ ] EC-07 `[정의됨]` 임베딩 생성 실패 → 502, row 미생성
- [ ] EC-08 `[정의됨]` 페이지 본문 없음(SPA/JS 렌더) → title 단독 임베딩 + weak-vector 경고 로그
- [ ] EC-09 `[확인 필요·낮음]` 302/301 리다이렉트 체인 URL → 원본 URL 그대로 저장
- [ ] EC-10 `[확인 필요·낮음]` non-http 스킴 탭에서 저장 시도 → 버튼 상태 미정의

### SC-05 — 자연어 검색

- [ ] EC-11 `[정의됨]` 빈 문자열 검색 → 400
- [ ] EC-12 `[정의됨]` 50자 초과 → 400
- [ ] EC-13 `[정의됨]` SQL 인젝션 패턴 → 파라미터 바인딩, 인젝션 불가
- [ ] EC-14 `[정의됨]` 검색 결과 0건 → 빈 상태 UI
- [ ] EC-15 `[확인 필요]` 필터 조합 + 검색 결과 0건 → 원인 구분 UX 미정의
- [ ] EC-16 `[확인 필요·낮음]` 이모지/한자 검색어 → 지원 범위 실측 없음

### SC-07 — 파일 임포트

- [ ] EC-17 `[정의됨]` 5MB 초과 파일 → 413
- [ ] EC-18 `[확인 필요]` 500개 초과 항목 → skipped 카운트, UI 노출 여부 미정의
- [ ] EC-19 `[정의됨]` 잘못된 MIME/확장자 위장 → 400
- [ ] EC-20 `[정의됨]` 파일 파싱 자체 실패(깨진 HTML/CSV) → 400
- [ ] EC-21 `[확인 필요·중간]` 업로드 중 취소 → 롤백 여부 미정의
- [ ] EC-22 `[확인 필요]` 파일 내부 중복 URL → 스킵/실패목록 처리 방식 미정의
- [ ] EC-23 `[확인 필요]` 소스 선택-실제 파일 형식 불일치 → 재선택 안내 여부 미정의

### SC-08 — 편집/삭제

- [ ] EC-24 `[정의됨]` 잘못된 id 형식 → 400
- [ ] EC-25 `[정의됨]` 존재하지 않는/타인 소유 id → RLS 필터 후 404
- [ ] EC-26 `[정의됨]` tags 11개 이상 → 400
- [ ] EC-27 `[정의됨]` tag 1개 51자 이상 → 400
- [ ] EC-28 `[정의됨]` description 2000자 초과 → 400
- [ ] EC-29 `[확인 필요·높음]` 삭제 직후 동일 카드 더블클릭(레이스) → 두 번째 요청 처리 미정의

### SC-09 — 죽은 링크 감지

- [ ] EC-31 `[정의됨]` 404/410 → `is_dead=true`
- [ ] EC-32 `[정의됨]` HEAD 미지원 서버 → GET 폴백
- [ ] EC-33 `[정의됨]` 과거 저장분 → 자동 소급 없음, 수동 backfill

### SC-11 — 탈퇴/인증

- [ ] EC-35 `[정의됨]` 탈퇴 API 500(부분 실패) → CASCADE 원자적 처리로 전체 롤백 추정
- [ ] EC-36 `[확인 필요]` 익스텐션 로그인 상태에서 웹앱 탈퇴 → 즉시 반영 여부 미정의

### 크로스커팅 (전 시나리오 공통)

- [ ] EC-37 `[확인 필요]` 세션 만료 상태로 API 호출 → 화면별 재로그인 리다이렉트 일관성 미확인
- [ ] EC-38 `[정의됨]` RLS 우회 시도 → 원천 차단, 404/빈 배열
- [ ] EC-39 `[확인 필요]` 네트워크 끊김 중 저장 버튼 연타 → debounce/disable 여부 미정의

## 5. 입력 검증 기준값 (참고, `front/lib/schemas.ts` 근거)

| 필드             | 제약                          | 위반 시 |
| ---------------- | ----------------------------- | ------- |
| `title`          | 1~500자                       | 400     |
| `url`            | `z.url()` 형식                | 400     |
| `content`        | 0~2000자                      | 400     |
| 검색 `query`     | 1~50자                        | 400     |
| `tags`           | 배열 최대 10개, 항목당 1~50자 | 400     |
| `category`       | 1~50자                        | 400     |
| `description`    | 최대 2000자                   | 400     |
| 임포트 파일 크기 | 최대 5MB                      | 413     |
| 임포트 항목 수   | 최대 500개(초과분 skip)       | —       |
| 썸네일 이미지    | 최대 5MB                      | 502     |
| `(user_id, url)` | UNIQUE                        | 409     |
