## 1. 요약

북마크 AI 관리 서비스는 브라우저 익스텐션과 웹앱을 결합하여, 사용자가 저장한 북마크에 AI가 자동으로 태그·카테고리를 부여하고 자연어 검색을 가능하게 하는 개인 지식 관리 도구다. "제목만 저장하고 잊어버리는" 북마크 문제를 AI 임베딩 기반 검색으로 해결하여, 사용자가 언제든 원하는 콘텐츠를 빠르게 다시 찾을 수 있게 한다.

## 3. 배경

### 이 이니셔티브는 무엇인가

브라우저 북마크 기능은 20년 이상 거의 변하지 않았다. 사람들은 매일 수십 개의 링크를 저장하지만, 저장 후 다시 찾는 비율은 현저히 낮다. CHI 2021 연구("Out of Sight and Out of Mind", 참가자 50명)는 이를 실증했다. 대부분의 사용자가 북마크를 저장해두고도 실제 검색에는 쓰지 않으며, 대신 검색엔진을 다시 쓰거나 히스토리를 뒤진다. 정보 재발견(re-finding) 연구에 따르면 웹 방문의 44%, 웹 검색의 33%가 이전 방문 페이지의 재방문이다. 즉, 재발견 수요는 실재하지만 현재 도구가 이를 제대로 지원하지 못하고 있다.

북마크 폴더 구조는 직접 분류해야 하고 내용이 아닌 제목만 저장되기 때문에 검색 자체가 불가능하다. 결국 중복 저장이 누적되고, 수백 개의 북마크가 쌓인 채 방치된다.

이 서비스는 AI를 활용해 저장 시 자동 태깅·분류를 수행하고, 벡터 임베딩으로 자연어 검색을 지원하여 "저장 → 재발견" 사이클을 완성한다.

### 왜 지금인가

- **AI 비용 급감**: GPT-4o-mini 출시(2024)로 태깅 비용이 유저당 월 $0.02 수준으로 낮아졌다. 2년 전 GPT-4 기준 유저당 $2.00이었던 비용이 100분의 1로 떨어져, 수익화 없이도 서비스 운영이 가능해졌다.
- **pgvector 성숙**: Supabase에서 pgvector 0.7+ 기반 벡터 검색을 별도 인프라 없이 제공한다. 별도 벡터 DB(Pinecone 등)를 도입하지 않아도 의미 기반 검색이 가능해졌다.
- **개인 지식 관리 수요 증가**: Notion, Obsidian 등 PKM(Personal Knowledge Management) 도구 사용자가 급증하고 있으며, 브라우저 북마크를 이 생태계에 통합하려는 니즈가 확인됐다. 신경과학 연구는 현대인이 정보 과부하로 인한 결정 피로를 겪고 있음을 보여준다. 북마크를 어느 폴더에 넣을지 결정하는 행위 자체도 인지 자원을 소모하는데, AI가 이 분류 결정을 대신하면 사용자의 인지 부하를 실질적으로 낮출 수 있다.
- **국내 경쟁 공백**: Mozilla Pocket이 2025년 7월 종료되며 국내 대안이 부재하다. Raindrop.io(해외)가 유사하지만 자연어 검색과 한국어 최적화가 없다. 국내에 무료·AI·자연어 검색을 모두 충족하는 서비스는 현재 존재하지 않는다.

## 4. 목표

### 왜 중요한가

- **사용자 관점**: 저장한 정보를 실제로 활용할 수 있게 되면, 브라우징 → 저장 → 검색 → 재사용의 지식 루프가 완성된다. 정보 과부하 시대에 개인 생산성 도구로서 실질적 가치를 제공한다.
- **회사 관점**: SaaS 구독 모델로 초기 무료 → 프리미엄 전환(freemium)을 통해 반복 수익을 만든다. 유저당 비용이 매우 낮아($0.02/월) 고마진 구조가 가능하다.
- **전략 부합**: 개인 AI 도구 시장 진입 → 이후 팀 협업/공유 기능 확장으로 B2B SaaS로의 전환 경로를 열 수 있다.

## 5. 시장 세그먼트

### 주요 타겟

| 세그먼트            | 특성                                                                                   |
| ------------------- | -------------------------------------------------------------------------------------- |
| **지식 노동자**     | 개발자, 디자이너, 리서처 — 하루 10개 이상 링크를 저장하며 나중에 다시 찾는 경우가 잦음 |
| **콘텐츠 큐레이터** | 뉴스레터 발행자, 블로거 — 리서치 소스 관리가 핵심 업무                                 |
| **학생·연구자**     | 논문, 아티클 저장 후 논거 찾기 용도로 활용                                             |

### 얼리 어답터

- Chrome 익스텐션에 익숙한 개발자/디자이너 (한국 기준 30~40대 직장인)
- Notion, Obsidian 등 PKM 도구 기존 사용자
- 북마크 폴더가 100개 이상인 "저장 강박" 사용자

### 제외 타겟

- 브라우저 북마크를 거의 사용하지 않는 모바일 우선 사용자
- Safari/Firefox 전용 사용자 (MVP는 Chrome 전용)
- 팀 단위 콘텐츠 공유가 주목적인 사용자 (v2 이후)

### 제약

- Chrome 브라우저 사용자에 한정 (Manifest V3 기준)
- 한국어/영어 혼용 콘텐츠 태깅 품질에 의존
- 모바일 앱 없음 (MVP 기준)

## 6. 가치 제안

### 어떤 고객의 Job을 해결하는가

"나중에 읽으려고 저장했는데 어디 뒀는지 기억이 안 난다" — 정보를 저장하는 행위 자체보다, **저장한 정보를 다시 활용하는 것**이 핵심 Job이다.

### 고객이 얻는 것

| 항목               | 내용                                                                                |
| ------------------ | ----------------------------------------------------------------------------------- |
| **시간 절약**      | 태그 분류를 직접 할 필요 없음. 저장 1.5초 내 자동 태그 0~3개 생성                   |
| **정보 재발견**    | "리액트 훅 관련 글" 같은 자연어로 검색, 관련 북마크를 유사도 순 반환                |
| **중복 저장 방지** | 같은 URL 백엔드 dedup (`(user_id,url)` UNIQUE + upsert, MVP). 사용자 알림 UI는 v1.1 |
| **정리 부담 제거** | 폴더 구조 없이 태그·카테고리로 자동 정리                                            |
| **개인 인사이트**  | 내가 무엇에 관심 갖는지 카테고리·태그·활동량 기반 시각화 (프리미엄)                 |

### 피하는 고통

| 기존 고통                     | 해결 방식                                     |
| ----------------------------- | --------------------------------------------- |
| 제목만 저장 → 내용 기억 못 함 | 본문 앞 2000자를 임베딩해 의미 기반 검색 가능 |
| 폴더 분류 직접 해야 함        | AI가 자동 카테고리 1개 + 태그 0~3개 부여      |
| 오래된 북마크 방치            | 카테고리/태그 필터로 재탐색 동선 제공         |

### 경쟁자보다 더 잘 해결하는 문제

| 경쟁자                   | 한계                                   | 우리의 차별점                                 |
| ------------------------ | -------------------------------------- | --------------------------------------------- |
| **브라우저 기본 북마크** | 폴더 수동 분류, 검색 불가              | AI 자동 태그 + 자연어 검색                    |
| **Raindrop.io**          | 자연어 검색 미지원, 한국어 최적화 없음 | pgvector 기반 의미 검색, 한국어 콘텐츠 최적화 |
| **Pocket**               | 태그 수동 입력, AI 없음                | 저장 즉시 AI 자동 분류                        |
| **Notion Web Clipper**   | Notion 내부에서만 검색 가능            | 별도 웹앱 + 익스텐션으로 빠른 접근            |

## 7. 솔루션

### 7.1 핵심 사용자 흐름

#### 랜딩 흐름 (비로그인 방문자 — A39)

```
1. 공개 랜딩 페이지(/welcome) 진입 (PUBLIC_PATHS)
2. 서비스 소개 3섹션 노출 (ServiceFeatures 공용 컴포넌트)
3. 서버에서 인증 상태 확인 후 CTA 분기
   - 비로그인 → 로그인 페이지
   - 로그인 → 대시보드(홈)
```

#### 온보딩 흐름 (신규 가입자)

```
1. Google OAuth 로그인 완료
2. 온보딩 화면 진입
   a. 익스텐션 설치 안내 (Chrome 웹스토어 링크)
   b. 첫 북마크 저장 유도 (데모 GIF)
3. 첫 북마크 저장 완료 시 메인 화면으로 이동 (Aha 모멘트)
4. 이후 대시보드 헤더 '사용법' 버튼으로 안내 모달 재진입 가능 (A39)
```

#### 저장 흐름 (익스텐션)

```
1. 사용자가 읽고 싶은 페이지에서 Cmd+Shift+S 또는 익스텐션 버튼 클릭
2. 익스텐션이 현재 탭의 제목, URL, 본문 앞 2000자를 수집
3. POST /api/bookmarks 로 서버 전송
4. 서버에서:
   a. GPT-4o-mini → 태그 0~3개(대→중→소 계층) + 카테고리 1개 생성
   b. text-embedding-3-small → 벡터 임베딩(1536차원) 생성
   c. Supabase에 저장 (원문 텍스트는 저장하지 않음)
5. 익스텐션 팝업에 "저장됨 + 자동 생성된 태그" 토스트 표시 (~1.5초)
```

#### URL 직접 추가 흐름 (웹앱)

```
1. 헤더 우측 "북마크 추가" 버튼 클릭
2. URL 입력 모달 팝업
   a. URL 형식 validation (URL 형태만 허용)
   b. "추가" 버튼 클릭 → POST /api/bookmarks
   c. "닫기" 버튼으로 모달 닫기
3. 저장 흐름 동일 진행 (AI 태깅 + 임베딩)
```

#### 파일 임포트 흐름 (웹앱 — WEB-14)

```
1. 헤더 우측 "파일 업로드" 버튼 클릭 → 파일 임포트 페이지 이동
2. 드래그앤드롭 또는 파일 선택 버튼으로 HTML 파일 업로드
   - 선택 후: 파일명·크기 표시 + 업로드 버튼 노출 (선택 파일 정보)
3. 파일 검증 (스피너 + 파일 검사 중)
4. 업로드 진행 (프로그레스 바, 예: 500/1000개, 취소 버튼)
5. AI 태깅 처리 (450/500 태깅 완료)
6. 완료 상태 분기:
   - 성공: 완료 메시지 + 홈 이동 버튼
   - 실패: 에러 메시지 + 재시도 버튼
   - 부분 완료: 경고 메시지 + 실패 목록 (v1.1)
```

#### 검색 흐름 (웹앱)

```
1. 웹앱(app.bookmark-ai.io)에서 검색창에 자연어 입력
   예: "머신러닝 입문 자료", "디자인 참고할 사이트"
2. POST /api/search 로 쿼리 전송
3. 서버에서:
   a. 쿼리를 임베딩으로 변환
   b. pgvector cosine similarity 검색 (사용자 북마크 중 상위 20개)
4. 결과: 제목, URL, 태그, 카테고리, 유사도 점수 반환
5. 사용자가 클릭 → 원본 페이지 이동
```

#### 탐색 흐름 (웹앱)

```
1. 웹앱 홈: 최근 저장 북마크 목록 표시 (그리드 뷰 default)
2. 사이드바 탭 선택: 전체 북마크 | 즐겨찾기 | 카테고리
3. 카테고리 클릭 → 해당 카테고리 북마크 필터링 표시
4. 정렬: 최신순(default) / 유사도순
5. 뷰 전환: 그리드 뷰(default) / 리스트 뷰 / 컴팩트 뷰 (뷰 3종, A50)
```

### 7.2 핵심 기능 상세

#### A기능: AI 자동 태깅

- **동작**: 저장 시 본문 앞 2000자를 GPT-4o-mini에 전달, `{ "tags": [{ "tag": "...", "confidence": 0~1 }, ...] }` JSON 응답. confidence 0.6 미만 태그는 자동 제외(정밀도 우선).
- **태그 수**: 0~3개 (대→중→소 계층, 분류 불가 페이지는 0개). 분류 체계는 `docs/specs/tag-taxonomy.md` — 대분류 13종(개발·AI/ML·디자인·비즈니스·학습·쇼핑·커뮤니티·콘텐츠·브랜드·게임·라이프스타일·여행·금융)
- **카테고리**: 1개 (`tags[0]` 기준, 유저별 개인 카테고리에 자동 생성·매핑 — 전역 사전 정의 목록 없음. migration 0004). `tags[0]`이 고정 13개 대분류에 없거나 태깅 실패(`tags=[]`)면 `category_id` NULL → 사이드바에서 **'미분류'** 한 항목으로 묶음(A47)
- **품질 측정**: 골든셋(`front/eval/tag-golden.json`, n=115) 대비 precision/recall/F1 + 대분류 정확도 회귀 게이트(`RUN_TAG_EVAL=1`, baseline F1 0.82). 실측 macro-F1 0.85·대분류 0.93(2026-07)
- **언어**: 한국어/영어 혼용 콘텐츠 모두 지원
- **원문 보존 여부**: 서버 처리 후 즉시 파기 — 저작권법 및 개인정보보호법 대응
- **사용자 수정**: 웹앱에서 태그 직접 편집 가능 (v1.1)

#### C기능: 개인 북마크 대시보드 (프리미엄 전용, v1.1)

- **목적**: 저장 행동 패턴을 시각화해 "나는 어떤 지식을 쌓고 있는가"를 보여주는 개인 인사이트 화면
- **구성 요소**:
  - 카테고리별 북마크 분포 — 도넛 차트 (개발 32%, 디자인 18% 등)
  - 주간 저장 활동 추이 — 막대 그래프 (최근 8주)
  - 자주 쓰인 태그 TOP 10 — 태그 클라우드 (크기 = 빈도)
  - 최다 저장 도메인 TOP 5 — 순위 리스트
  - 월별 활동 히트맵 — GitHub contribution 스타일
- **집계 기준**: 사용자 본인 북마크만. 타인 데이터 없음
- **기간 선택**: 최근 30일 / 90일 / 전체 (드롭다운)
- **무료 제한**: 최근 7일 요약만 노출. 전체 기간·상세 차트는 프리미엄 전용
- **데이터 출처**: 기존 `bookmarks` 테이블 집계 쿼리만으로 구현. 별도 테이블 불필요

#### D기능: 즐겨찾기

- **동작**: 카드 내 즐겨찾기 버튼 클릭 → 즐겨찾기 토글
- **탐색**: 사이드바 "즐겨찾기" 탭에서 즐겨찾기된 북마크만 필터링 표시
- **카드 메뉴**: 수정(v1.1) / 삭제(MVP) — 카드 메뉴 버튼 → 모달

#### E기능: 파일 임포트 (WEB-14)

- **지원 형식**: 브라우저 북마크 HTML 파일
- **업로드 방식**: 드래그앤드롭 또는 파일 선택 버튼
- **처리 흐름**: 파일 검증 → 업로드 진행 → AI 태깅 처리 순차 진행
- **진행 상황**: 프로그레스 바 (처리 중 개수/전체 표시)
- **취소**: 업로드 중 취소 가능
- **상태 분기**: 성공 / 실패 / 부분 완료 각각 별도 UI 표시

#### B기능: 자연어 검색

- **동작**: 검색어를 text-embedding-3-small로 임베딩 후 pgvector cosine similarity 검색
- **검색창**: 최대 50자 입력 제한
- **최근 검색**: 최근 검색어 태그 형태 표시 (width 최대 200px, 초과 시 `...` 처리) — MVP
- **인덱스**: HNSW 인덱스 (pgvector 0.5+) — 대규모 북마크에서도 밀리초 단위 응답
- **반환 수**: 기본 20개, 유사도 임계값 0.5 이상만 표시
- **필터 조합**: 자연어 검색 + 태그/카테고리 필터 동시 적용 가능 (v1.1)
